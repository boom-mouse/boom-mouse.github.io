Java优缺点和JVM

Java继承了C和C++很多内容，可以看成类C的产物

优点：

  • 跨平台性:Java一开始最注重了移植和跨平台性，比如Java的int永远是32位，不想C++可能是16、32

    • 原理：只需要在运行的操作系统上安装Java虚拟机(JVM Java visual machine)
      • 如果有m种语言，n种操作系统，若没有虚拟机，则需要m*n种编译器，若有虚拟机在中间为操系统解释，则需要m+n种编译器

  • 面向对象性:非常适合大型软件的开发和设计。

  • 健壮性:吸收了C/C++的优点，去除了影响程序健壮性的部分(如指针、内存的申请与释放)

  • 安全性高:Java使用于分布式/网络环境

  • 简单性:Java就是C++语法的简化版，可以称为“C++--”

  • 高性能:最初确实有低性能的诟病；客观上高级语言运行效率总是低于低级语言。Java本身发展中通过虚拟机优化提升了效率，如通过JIT（just in time)即时编译技术提高运行效率。

缺点:

  • 语法复杂:语法过于严谨复杂，于python、php等相比入门难度较大

    • 一般适用于大型网站开发:整个框架较重，维护成本高

  • 并不适用所有领域:比如objective C，swift在ios的地位，JavaScript在浏览器中的处理，windows程序多用C或C++、C#。Java在服务器端编程和跨平台客户端应用领域很有优势

JVM

jvm是一个虚拟计算机，是Java的运行环境，且具有指令集，负责执行指令、管理数据

标识符规则

没什么说的和C等基本一致，但有业内规则需要注意，也为了可读性，以下不影响程序运行

  1. 包名：多单词组成时所有字母小写:helloworld
  2. 类名、接口名:多单词组成时所有单词首字母大写:HelloWorld
  3. 变量名、方法名:多单词组成时第一个单词首字母小写，第二个单词开始，所有单词首字母大写:helloWorldWorld
  4. 常量名:所有字母大写。多个单词时用下划线连接HELLO_WORLD

布尔类型boolean

布尔类型只有2个值true和false，常用于判断、循环语句

需要注意的是Java不同于C语言，它并不等于1/0，也就是if(1)是错误的写法

有说1字节的，有说和int类型一样8字节的。一般不讨论布尔的内存。

Amos-Chen的结论

结论

  1. boolean类型被编译为int类型，等于是说JVM里占用字节和int完全一样，int是4个字节，于是boolean也是4字节
  2. boolean数组在Oracle的JVM中，编码为byte数组，每个boolean元素占用8位=1字节
  3. （待考证）boolean数组在非Oracle JVM里面，占用字节数也许不是1字节
